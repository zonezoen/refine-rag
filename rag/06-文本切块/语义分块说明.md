# 语义分块详解

## 什么是语义分块？

语义分块（Semantic Chunking）是一种智能文本分割方法，它根据**语义相似度**而不是固定字符数来分割文本。

---

## 传统分块 vs 语义分块

### 传统分块（RecursiveCharacterTextSplitter）

```python
text = "黑神话悟空是一款动作游戏。游戏基于西游记改编。主角是孙悟空的转世。"

# 按 30 字符分割
chunk1 = "黑神话悟空是一款动作游戏。游戏基于西游"
chunk2 = "记改编。主角是孙悟空的转世。"
```

**问题**：
- ❌ 在"西游"中间断开
- ❌ 破坏了语义完整性

### 语义分块（SemanticChunker）

```python
# 根据语义相似度分割
chunk1 = "黑神话悟空是一款动作游戏。游戏基于西游记改编。"  # 游戏介绍
chunk2 = "主角是孙悟空的转世。"  # 角色介绍
```

**优势**：
- ✅ 保持语义完整
- ✅ 自动识别主题边界

---

## 工作原理

```
原始文本
    ↓
按句子分割
    ↓
计算相邻句子的 embedding
    ↓
计算余弦相似度
    ↓
相似度 < 阈值？
    ├─ 是 → 创建新块
    └─ 否 → 合并到当前块
```

### 示例

```python
句子1: "黑神话悟空是一款动作游戏。"
句子2: "游戏基于西游记改编。"
相似度: 0.85 (高) → 合并

句子2: "游戏基于西游记改编。"
句子3: "主角是孙悟空的转世。"
相似度: 0.65 (低) → 分割
```

## LangChain 语义分块器

### 基本用法

```python
from langchain_experimental.text_splitter import SemanticChunker
from langchain_huggingface import HuggingFaceEmbeddings

embeddings = HuggingFaceEmbeddings(
    model_name="BAAI/bge-small-zh-v1.5"
)

splitter = SemanticChunker(
    embeddings=embeddings,
    breakpoint_threshold_type="percentile",
    breakpoint_threshold_amount=90
)

chunks = splitter.split_documents(docs)
```

### 参数详解

#### 1. `breakpoint_threshold_type`

控制如何计算分割阈值：

**percentile（百分位数）**：推荐
```python
breakpoint_threshold_type="percentile"
breakpoint_threshold_amount=90  # 90% 不相似度时分割
```
- 适用于大多数场景
- 自动适应文档特点

**standard_deviation（标准差）**：
```python
breakpoint_threshold_type="standard_deviation"
breakpoint_threshold_amount=2  # 2个标准差
```
- 适用于相似度分布均匀的文档

**interquartile（四分位数）**：
```python
breakpoint_threshold_type="interquartile"
breakpoint_threshold_amount=1.5
```
- 适用于有异常值的文档

#### 2. `breakpoint_threshold_amount`

控制分割的敏感度：

| 值 | 效果 | 适用场景 |
|----|------|---------|
| 85 | 块很多（细粒度） | 精确检索 |
| 90 | 块适中（推荐） | 通用场景 |
| 95 | 块较少（粗粒度） | 长上下文 |

## LlamaIndex vs LangChain

| 特性 | LlamaIndex | LangChain |
|------|-----------|-----------|
| 分块器名称 | `SemanticSplitterNodeParser` | `SemanticChunker` |
| 参数 | `buffer_size`, `breakpoint_percentile_threshold` | `breakpoint_threshold_type`, `breakpoint_threshold_amount` |
| 返回类型 | `Node` 对象 | `Document` 对象 |
| 易用性 | 较复杂 | 简单 |
| 灵活性 | 更高 | 适中 |

## 性能对比

### 速度测试（1000字文档）

| 方法 | 时间 | 块数 |
|------|------|------|
| 传统分块 | 0.01秒 | 5块 |
| 语义分块 | 0.5秒 | 3块 |

**结论**：语义分块慢 50 倍，但质量更高

## 使用建议

### 适合语义分块的场景

✅ 问答系统（需要完整上下文）  
✅ 长文档分析（自动识别主题）  
✅ 高质量检索（语义相关性强）  
✅ 离线处理（可以接受较慢速度）

### 适合传统分块的场景

✅ 实时处理（需要快速响应）  
✅ 大规模文档（数千个文档）  
✅ 固定格式文档（如代码、表格）  
✅ 资源受限环境（无法运行 embedding 模型）

## 最佳实践

### 1. 混合策略

```python
# 先用传统分块粗分
coarse_chunks = RecursiveCharacterTextSplitter(
    chunk_size=2000
).split_documents(docs)

# 再用语义分块细分
fine_chunks = []
for chunk in coarse_chunks:
    fine_chunks.extend(
        SemanticChunker(embeddings).split_documents([chunk])
    )
```

### 2. 参数调优

```python
# 测试不同阈值
for threshold in [85, 90, 95]:
    splitter = SemanticChunker(
        embeddings=embeddings,
        breakpoint_threshold_amount=threshold
    )
    chunks = splitter.split_documents(docs)
    print(f"阈值 {threshold}: {len(chunks)} 块")
```

### 3. 缓存 Embedding

```python
# 避免重复计算
from functools import lru_cache

@lru_cache(maxsize=1000)
def get_embedding(text):
    return embeddings.embed_query(text)
```

## 常见问题

### Q1: 为什么语义分块这么慢？

A: 需要为每个句子计算 embedding，计算量大。可以：
- 使用更小的 embedding 模型
- 减少文档长度
- 使用 GPU 加速

### Q2: 如何选择阈值？

A: 从 90 开始，根据结果调整：
- 块太多 → 增加阈值（如 95）
- 块太少 → 降低阈值（如 85）

### Q3: 能用 OpenAI embedding 吗？

A: 可以，但成本较高：
```python
from langchain_openai import OpenAIEmbeddings
embeddings = OpenAIEmbeddings()
```

## 总结

语义分块是一种强大的文本分割方法，特别适合需要保持语义完整性的场景。虽然速度较慢，但在问答系统、文档分析等应用中，能显著提高检索质量和答案准确性。
