### 2. drop_properties 错误处理

Milvus 2.4 版本的 `drop_database_properties` 和 `drop_collection_properties` 存在已知 bug：
- 属性实际上已经被删除
- 但会返回错误信息："only support alter properties, but properties is empty"

**解决方案**：添加 try-except 处理

```python
try:
    client.drop_database_properties(
        db_name="my_database_1",
        property_keys=["database.max.collections"]
    )
    print("✓ 已删除属性")
except Exception as e:
    print("⚠️  删除属性时返回错误（这是 Milvus 2.4 的已知问题，属性实际已删除）")
```

**修复的文件**：
- `01-database.py` - 第 124-137 行
- `02-collection.py` - 第 74-79 行

### 3. 异常处理优化

移除了不存在的 `exceptions.AlreadyExistError`，改用通用异常处理：

```python
# 修复前
except exceptions.AlreadyExistError:
    print("已存在")

# 修复后
except Exception as e:
    if "already exist" in str(e):
        print("已存在")
    else:
        raise e
```

### 4. Alias 重复创建问题

在创建 alias 前先清理可能存在的旧 alias：

```python
# 先清理可能存在的旧 alias
for alias_name in ["alias3", "alias4"]:
    try:
        client.drop_alias(alias=alias_name)
        print(f"✓ 已删除旧的 {alias_name}")
    except Exception:
        pass  # alias 不存在，忽略

# 再创建新的 alias
client.create_alias(collection_name=new_collection_name, alias="alias3")
client.create_alias(collection_name=new_collection_name, alias="alias4")
```

**修复的文件**：
- `02-collection.py` - 第 140-149 行
