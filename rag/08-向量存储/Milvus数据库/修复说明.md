- 部署方式: Docker Standalone

## 已知问题和解决方案

### 1. BM25 函数问题

#### 问题描述
在 Milvus 2.4.15 中，使用 `Function` 和 `FunctionType.BM25` 时会遇到以下错误：
```
DataNotMatchException: Insert missed an field `sparse` to collection without set nullable==true or set default_value
```

#### 原因
- BM25 函数功能在 2.4.x 版本中还不够成熟
- Function 自动生成字段的机制可能不完全支持
- 需要手动提供 sparse 向量或使用其他方式

#### 解决方案

**方案 1：使用外部 BM25 库（推荐）**
```python
from rank_bm25 import BM25Okapi
import jieba

# 1. 准备文档
documents = [
    '信息检索是一个研究领域。',
    '信息检索专注于在大型数据集中查找相关信息。',
    '数据挖掘和信息检索在研究中有所重叠。',
]

# 2. 分词
tokenized_docs = [list(jieba.cut(doc)) for doc in documents]

# 3. 创建 BM25 模型
bm25 = BM25Okapi(tokenized_docs)

# 4. 搜索
query = "信息检索"
tokenized_query = list(jieba.cut(query))
scores = bm25.get_scores(tokenized_query)

# 5. 获取 top-k 结果
top_n = 3
top_indices = sorted(range(len(scores)), key=lambda i: scores[i], reverse=True)[:top_n]
```

**方案 2：等待 Milvus 2.5+ 版本**
- BM25 函数功能在更新的版本中可能会更稳定
- 升级到 Milvus 2.5 或更高版本

### 2. TEXT_MATCH 函数问题

#### 问题描述
```
invalid expression: TEXT_MATCH(text, 'text_1 text_2'): invalid parameter
```

#### 原因
- TEXT_MATCH 功能在 2.4.15 中可能需要特定配置
- 语法可能与文档不一致

#### 解决方案

**使用标准 SQL LIKE 语法（已修复）**
```python
# 替代 TEXT_MATCH
filter_expr = "text like 'text_1%' or text like 'text_2%'"
results = client.search(
    collection_name=COLLECTION_NAME,
    data=[query_vector],
    anns_field="vector",
    filter=filter_expr,
    limit=3,
    output_fields=["text"]
)
```

### 3. 索引参数兼容性

#### 稀疏向量索引
- 必须使用 `metric_type="IP"`（内积）
- 不支持 `metric_type="BM25"`
- 简化的参数配置：
```python
index_params.add_index(
    field_name="sparse",
    index_type="SPARSE_INVERTED_INDEX",
    metric_type="IP",
    params={"drop_ratio_build": 0.2}
)
```

## 推荐配置

### 对于全文搜索场景
1. 使用外部 BM25 库（rank-bm25）进行初步召回
2. 使用 Milvus 进行向量语义搜索
3. 结合两者的结果进行混合排序

### 对于文本过滤场景
1. 使用标准 SQL 表达式（LIKE, IN, ==）
2. 避免使用实验性的 TEXT_MATCH 功能
