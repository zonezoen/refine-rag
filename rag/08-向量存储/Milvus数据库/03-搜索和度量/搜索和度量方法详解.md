## 什么是向量搜索？

向量搜索是在高维向量空间中找到与查询向量最相似的向量的过程。

**通俗比喻：**
想象你在一个巨大的音乐库中找相似的歌曲。每首歌可以用一个向量表示（包含节奏、音调、风格等特征）。
当你输入一首喜欢的歌，系统会找到向量最接近的其他歌曲推荐给你。

**核心概念：**
- **查询向量**：你要搜索的目标（如用户输入的问题）
- **候选向量**：数据库中存储的所有向量
- **距离度量**：衡量两个向量相似程度的方法
- **Top-K**：返回最相似的 K 个结果

## 距离度量方式

### 1. L2 距离（欧几里得距离）

#### 原理
L2 距离就是两点之间的直线距离，和我们日常生活中测量距离的方式一样。

**数学公式：**
```
L2(A, B) = √[(a₁-b₁)² + (a₂-b₂)² + ... + (aₙ-bₙ)²]
```

**通俗比喻：**
就像在地图上测量两个城市之间的直线距离。距离越小，两个点越接近。

#### 特点
- ✅ 直观易懂，符合人类对"距离"的认知
- ✅ 考虑向量的大小（magnitude）
- ✅ 适合连续数值特征
- ❌ 对向量的尺度敏感（需要归一化）
- ❌ 高维空间中计算量大

#### 使用场景
- 图像特征比较（像素值、颜色直方图）
- 地理位置数据
- 物理测量数据
- 需要考虑向量大小的场景

#### 代码示例
```python
index_params.add_index(
    field_name="vector",
    metric_type="L2",  # 使用 L2 距离
    index_type="HNSW"
)

# 搜索时
results = client.search(
    collection_name=COLLECTION_NAME,
    data=[query_vector],
    anns_field="vector",
    limit=5,
    search_params={"metric_type": "L2"}
)
```


#### 实际例子
```python
# 两个向量
A = [1, 2, 3]
B = [4, 5, 6]

# L2 距离计算
L2 = √[(1-4)² + (2-5)² + (3-6)²]
   = √[9 + 9 + 9]
   = √27
   ≈ 5.196

# 距离越小越相似
```

### 2. IP 距离（内积）

#### 原理
IP (Inner Product) 是两个向量对应元素相乘后求和。值越大，向量越相似。

**数学公式：**
```
IP(A, B) = a₁×b₁ + a₂×b₂ + ... + aₙ×bₙ
```

**通俗比喻：**
像两个人的兴趣匹配度。每个维度代表一个兴趣领域的强度，相乘后求和得到总体匹配度。
如果两人在同一领域都很感兴趣（值都大），匹配度就高。

#### 特点
- ✅ 计算速度快
- ✅ 适合已归一化的向量
- ✅ 可以表示方向和大小的综合相似度
- ❌ 不满足三角不等式（不是真正的"距离"）
- ❌ 对向量大小敏感

#### 使用场景
- 推荐系统（用户-物品匹配）
- 协同过滤
- 已归一化的嵌入向量
- 需要考虑向量大小的相似度计算

#### 代码示例
```python
index_params.add_index(
    field_name="vector",
    metric_type="IP",  # 使用内积
    index_type="HNSW"
)

# 搜索时返回内积最大的结果
results = client.search(
    collection_name=COLLECTION_NAME,
    data=[query_vector],
    anns_field="vector",
    limit=5,
    search_params={"metric_type": "IP"}
)
```

#### 实际例子
```python
# 两个向量
A = [1, 2, 3]
B = [4, 5, 6]

# 内积计算
IP = 1×4 + 2×5 + 3×6
   = 4 + 10 + 18
   = 32

# 值越大越相似
```

### 3. COSINE 距离（余弦相似度）

#### 原理
COSINE 测量两个向量之间的夹角，只关注方向，不关注大小。

**数学公式：**
```
COSINE(A, B) = (A·B) / (||A|| × ||B||)
```

**通俗比喻：**
像比较两篇文章的主题相似度。不管文章长短（向量大小），只看主题方向是否一致。
两篇都讨论"人工智能"的文章，即使一篇很长一篇很短，主题相似度也很高。

#### 特点
- ✅ 对向量大小不敏感（自动归一化）
- ✅ 适合文本、语义相似度
- ✅ 值域固定在 [-1, 1] 或 [0, 1]
- ✅ 高维稀疏向量效果好
- ❌ 计算稍慢（需要归一化）

#### 使用场景
- 文本相似度（TF-IDF、词嵌入）
- 语义搜索
- 文档检索
- 推荐系统（只关注偏好方向）
- NLP 任务

#### 代码示例
```python
import numpy as np

# 归一化向量（COSINE 需要）
def normalize_vector(vector):
    norm = np.linalg.norm(vector)
    return vector / norm if norm != 0 else vector

query_vector = normalize_vector([1, 2, 3])

index_params.add_index(
    field_name="vector",
    metric_type="COSINE",  # 使用余弦相似度
    index_type="HNSW"
)

results = client.search(
    collection_name=COLLECTION_NAME,
    data=[query_vector],
    anns_field="vector",
    limit=5,
    search_params={"metric_type": "COSINE"}
)
```

#### 实际例子
```python
# 两个向量
A = [1, 2, 3]
B = [2, 4, 6]  # B 是 A 的两倍

# L2 距离
L2(A, B) = √[(1-2)² + (2-4)² + (3-6)²] = √14 ≈ 3.74

# 余弦相似度
COSINE(A, B) = 1.0  # 完全相似（方向相同）

# 可见 COSINE 只关注方向，不关注大小
```

---

### 度量方式对比

| 度量方式 | 计算复杂度 | 归一化需求 | 适用数据类型 | 典型应用 | 值域 |
|---------|-----------|-----------|------------|---------|------|
| **L2** | 中等 | 建议归一化 | 连续数值 | 图像、物理测量 | [0, +∞) |
| **IP** | 低 | 必须归一化 | 连续数值 | 推荐系统 | (-∞, +∞) |
| **COSINE** | 高 | 自动归一化 | 文本、稀疏向量 | NLP、文档检索 | [-1, 1] |

**选择建议：**
```
开始
 ↓
数据类型是文本/语义？
 ├─ 是 → 使用 COSINE
 └─ 否 ↓
    向量已归一化？
     ├─ 是 → 使用 IP（最快）
     └─ 否 → 使用 L2（最通用）
```

## 搜索方法详解

### 1. 基本 ANN 搜索

#### 原理
ANN (Approximate Nearest Neighbor) 是最基础的向量搜索，找到与查询向量最相似的 Top-K 个结果。

**通俗比喻：**
在图书馆找最相关的 5 本书。输入你的需求，系统返回最匹配的 5 本。

#### 特点
- ✅ 最常用的搜索方式
- ✅ 支持单向量和批量搜索
- ✅ 可以指定返回字段
- ✅ 速度快

#### 使用场景
- 语义搜索
- 图像检索
- 推荐系统
- 相似商品查找

#### 代码示例
```python
# 单向量搜索
query_vector = [0.1, 0.2, ..., 0.5]
results = client.search(
    collection_name="my_collection",
    data=[query_vector],
    anns_field="vector",
    limit=5,  # 返回 Top-5
    output_fields=["title", "content"]  # 返回这些字段
)

# 批量搜索（一次搜索多个查询）
query_vectors = [[0.1, 0.2, ...], [0.3, 0.4, ...]]
results = client.search(
    collection_name="my_collection",
    data=query_vectors,  # 多个查询向量
    anns_field="vector",
    limit=5
)

# 遍历结果
for hits in results:
    for hit in hits:
        print(f"ID: {hit['id']}, 距离: {hit['distance']}")
```

#### 性能优化
- 使用批量搜索提高吞吐量
- 合理设置 `limit` 值（不要过大）
- 只返回需要的字段（`output_fields`）

### 2. 过滤搜索

#### 原理
在向量搜索的基础上，添加元数据过滤条件，只在满足条件的向量中搜索。

**通俗比喻：**
在图书馆找书，但只在"2020年后出版"且"评分>4.5"的书中找。

#### 两种过滤模式

**标准过滤（Pre-filtering）：**
先过滤，再搜索。适合过滤后数据量较大的场景。

**迭代过滤（Post-filtering）：**
先搜索，再过滤。适合过滤条件很严格的场景。

#### 特点
- ✅ 结合向量相似度和业务规则
- ✅ 支持复杂的过滤表达式
- ✅ 提高结果的相关性
- ❌ 可能降低搜索速度

#### 使用场景
- 电商搜索（价格、品牌、评分过滤）
- 内容推荐（时间、分类、标签过滤）
- 文档检索（作者、日期、类型过滤）

#### 代码示例
```python
# 标准过滤
results = client.search(
    collection_name="products",
    data=[query_vector],
    anns_field="vector",
    limit=10,
    filter='price < 1000 and rating > 4.5 and category == "electronics"',
    output_fields=["name", "price", "rating"]
)

# 迭代过滤（适合严格过滤条件）
results = client.search(
    collection_name="products",
    data=[query_vector],
    anns_field="vector",
    limit=10,
    search_params={
        "metric_type": "L2",
        "hints": "iterative_filter"  # 启用迭代过滤
    },
    filter='rare_category == "limited_edition"',
    output_fields=["name"]
)
```

#### 过滤表达式语法
```python
# 比较运算符
"price > 100"
"rating >= 4.5"
"stock != 0"

# 逻辑运算符
"price < 1000 and rating > 4.0"
"category == 'A' or category == 'B'"

# 字符串匹配
"name like 'iPhone%'"  # 以 iPhone 开头
"description like '%wireless%'"  # 包含 wireless

# 范围查询
"price in [100, 200, 300]"
"id not in [1, 2, 3]"

# 组合条件
"(price < 1000 and rating > 4.5) or (brand == 'Apple')"
```

### 3. 范围搜索

#### 原理
不是找 Top-K 个最相似的，而是找距离在某个范围内的所有向量。

**通俗比喻：**
不是找"最近的 5 家餐厅"，而是找"距离我 1-3 公里之间的所有餐厅"。

#### 特点
- ✅ 返回所有满足距离条件的结果
- ✅ 适合需要阈值判断的场景
- ✅ 结果数量不固定
- ❌ 可能返回大量结果

#### 使用场景
- 相似度阈值判断（如人脸识别）
- 异常检测（距离过大的为异常）
- 聚类分析
- 质量控制

#### 代码示例
```python
# L2 距离范围搜索
results = client.search(
    collection_name="faces",
    data=[query_vector],
    anns_field="vector",
    limit=100,  # 最多返回 100 个
    search_params={
        "metric_type": "L2",
        "params": {
            "radius": 1.0,        # 外圈半径（最大距离）
            "range_filter": 0.5   # 内圈半径（最小距离）
        }
    },
    output_fields=["name"]
)

# 返回距离在 [0.5, 1.0] 之间的所有向量
```

#### 距离范围说明
```
L2 距离：range_filter < distance <= radius
IP 距离：radius <= distance < range_filter
COSINE：radius <= distance < range_filter

注意：L2 的范围是反的！
```

### 4. 分组搜索

#### 原理
按某个字段分组，每组返回最相似的结果。避免结果被某一组支配。

**通俗比喻：**
搜索"好看的电影"，不想结果全是"漫威电影"，而是希望每个类型（动作、喜剧、科幻）都返回几部。

#### 特点
- ✅ 结果多样性更好
- ✅ 避免结果集中在某一类
- ✅ 适合多文档/多来源场景
- ❌ 搜索速度稍慢

#### 使用场景
- RAG 系统（每个文档返回最相关的几个片段）
- 多样化推荐
- 新闻聚合（每个来源返回几条）
- 商品搜索（每个品牌返回几个）

#### 代码示例
```python
# 基本分组搜索
results = client.search(
    collection_name="documents",
    data=[query_vector],
    anns_field="vector",
    limit=5,  # 返回 5 个不同的组
    group_by_field="doc_id",  # 按文档 ID 分组
    output_fields=["doc_id", "chunk_text"]
)

# 配置每组大小
results = client.search(
    collection_name="documents",
    data=[query_vector],
    anns_field="vector",
    limit=3,  # 返回 3 个不同的文档
    group_by_field="doc_id",
    group_size=2,  # 每个文档返回 2 个最相关的片段
    strict_group_size=True,  # 严格保证每组有 2 个结果
    output_fields=["doc_id", "chunk_text"]
)
```

#### 实际应用示例
```python
# RAG 场景：从多个文档中检索
# 假设有 100 个文档，每个文档切分成多个片段
# 目标：从 5 个最相关的文档中，每个文档取 3 个最相关片段

results = client.search(
    collection_name="knowledge_base",
    data=[query_embedding],
    anns_field="embedding",
    limit=5,  # 5 个文档
    group_by_field="document_id",
    group_size=3,  # 每个文档 3 个片段
    output_fields=["document_id", "chunk_text", "page_number"]
)

# 结果：最多 5×3=15 个片段，来自 5 个不同文档
```

### 5. 全文搜索（BM25）

#### 原理
BM25 是一种基于词频的文本检索算法，不依赖向量，直接在文本上进行关键词匹配和排序。

**通俗比喻：**
传统的搜索引擎方式。输入"人工智能"，找到包含这些关键词的文档，并根据词频、文档长度等因素排序。

#### BM25 算法特点
- 考虑词频（TF）：词出现越多，相关性越高
- 考虑逆文档频率（IDF）：越罕见的词权重越高
- 考虑文档长度：避免长文档占优势
- 不需要向量化，直接文本匹配

#### 特点
- ✅ 精确的关键词匹配
- ✅ 不需要预先生成向量
- ✅ 适合关键词搜索
- ✅ 可以与向量搜索混合使用
- ❌ 不理解语义（"汽车"和"车辆"是不同的词）

#### 使用场景
- 关键词搜索
- 文档检索
- 混合搜索（BM25 + 向量搜索）
- 需要精确匹配的场景

#### 代码示例
```python
from pymilvus import Function, FunctionType

# 1. 创建 schema 时定义 BM25 函数
schema = client.create_schema()
schema.add_field(field_name="id", datatype=DataType.INT64, is_primary=True, auto_id=True)
schema.add_field(
    field_name="text", 
    datatype=DataType.VARCHAR, 
    max_length=1000, 
    enable_analyzer=True  # 启用文本分析
)
schema.add_field(field_name="sparse", datatype=DataType.SPARSE_FLOAT_VECTOR)

# 定义 BM25 函数
bm25_function = Function(
    name="text_bm25_emb",
    input_field_names=["text"],
    output_field_names=["sparse"],
    function_type=FunctionType.BM25,
)
schema.add_function(bm25_function)

# 2. 创建索引
index_params = client.prepare_index_params()
index_params.add_index(
    field_name="sparse",
    index_type="SPARSE_INVERTED_INDEX",
    metric_type="BM25",
    params={
        "bm25_k1": 1.2,  # 词频饱和参数
        "bm25_b": 0.75   # 文档长度归一化参数
    }
)

# 3. 搜索
query_text = "人工智能"
results = client.search(
    collection_name="documents",
    data=[query_text],
    anns_field="sparse",
    limit=5,
    output_fields=["text"]
)
```

#### BM25 参数说明
```python
# k1: 控制词频的影响
# - 值越大，词频的影响越大
# - 推荐范围：1.2 - 2.0
# - 默认：1.2

# b: 控制文档长度归一化
# - 0 = 不考虑文档长度
# - 1 = 完全归一化
# - 推荐范围：0.5 - 0.8
# - 默认：0.75
```

### 6. 文本匹配搜索

#### 原理
在向量搜索的基础上，添加文本匹配过滤，结合语义相似度和关键词匹配。

**通俗比喻：**
既要找"意思相近"的文档（向量搜索），又要确保包含特定关键词（文本匹配）。

#### 特点
- ✅ 结合语义和关键词
- ✅ 提高结果精确度
- ✅ 支持多词匹配
- ❌ 需要启用文本分析器

#### 使用场景
- 混合搜索（语义 + 关键词）
- 需要精确关键词的语义搜索
- 合规性检查（必须包含某些词）

#### 代码示例
```python
# 1. 创建 schema 时启用文本匹配
schema.add_field(
    field_name="text",
    datatype=DataType.VARCHAR,
    max_length=1000,
    enable_analyzer=True,  # 启用文本分析
    enable_match=True      # 启用文本匹配
)

# 2. 搜索时使用 TEXT_MATCH 过滤
results = client.search(
    collection_name="documents",
    data=[query_vector],
    anns_field="vector",
    filter="TEXT_MATCH(text, 'machine learning artificial intelligence')",
    limit=5,
    output_fields=["text"]
)

# 匹配包含 "machine learning" 或 "artificial intelligence" 的文档
```

#### TEXT_MATCH 语法
```python
# 单词匹配（OR 逻辑）
"TEXT_MATCH(text, 'word1 word2')"  # 包含 word1 或 word2

# 短语匹配
"TEXT_MATCH(text, '\"exact phrase\"')"  # 精确短语

# 结合其他过滤条件
"TEXT_MATCH(text, 'AI ML') and category == 'tech'"
```

### 7. 迭代搜索

#### 原理
使用迭代器分批获取大量搜索结果，避免一次性加载过多数据导致内存溢出。

**通俗比喻：**
像翻书一样，一页一页地看，而不是一次把整本书都复印下来。

#### 特点
- ✅ 可以获取大量结果（超过默认限制）
- ✅ 内存占用低
- ✅ 支持流式处理
- ❌ 速度比一次性搜索慢

#### 使用场景
- 需要获取大量结果（>1000）
- 数据导出
- 批量处理
- 内存受限环境

#### 代码示例
```python
# 创建搜索迭代器
iterator = client.search_iterator(
    collection_name="large_collection",
    data=[query_vector],
    anns_field="vector",
    batch_size=1000,  # 每批 1000 条
    limit=50000,      # 总共获取 50000 条
    output_fields=["title", "content"]
)

# 迭代获取结果
all_results = []
while True:
    batch = iterator.next()
    if not batch:
        iterator.close()
        break
    
    # 处理这一批结果
    for hit in batch:
        all_results.append(hit.to_dict())
        # 或者直接处理，不存储
        process_result(hit)

print(f"总共获取 {len(all_results)} 条结果")
```

#### 性能优化
```python
# 合理设置 batch_size
# - 太小：请求次数多，开销大
# - 太大：内存占用高
# 推荐：1000 - 5000

# 流式处理，不存储所有结果
while True:
    batch = iterator.next()
    if not batch:
        break
    
    # 直接处理，不累积
    for hit in batch:
        save_to_database(hit)  # 立即保存
        # 不要 all_results.append(hit)
```

### 8. 元数据查询

#### 原理
不基于向量相似度，而是直接根据元数据字段查询，类似传统数据库的 SQL 查询。

**通俗比喻：**
不是找"相似的书"，而是直接查"作者是张三的所有书"。

#### 三种查询方式

**Get 方法：根据 ID 查询**
```python
# 查询指定 ID 的记录
results = client.get(
    collection_name="documents",
    ids=[1, 2, 3, 4, 5],
    output_fields=["title", "content", "author"]
)

for result in results:
    print(f"ID: {result['id']}, 标题: {result['title']}")
```

**Query 方法：条件查询**
```python
# 根据条件查询
results = client.query(
    collection_name="documents",
    filter="author == 'John' and year > 2020",
    output_fields=["title", "year"],
    limit=10
)

for result in results:
    print(f"标题: {result['title']}, 年份: {result['year']}")
```

**QueryIterator：分页查询**
```python
from pymilvus import connections, Collection

connections.connect(uri="http://localhost:19530")
collection = Collection("documents")

# 创建查询迭代器
iterator = collection.query_iterator(
    batch_size=100,
    expr="category == 'tech'",
    output_fields=["title", "author"]
)

# 迭代获取结果
while True:
    batch = iterator.next()
    if not batch:
        iterator.close()
        break
    
    for item in batch:
        print(f"标题: {item['title']}")
```

#### 使用场景
- 精确查找（根据 ID）
- 元数据过滤
- 数据导出
- 统计分析

## 其他常用搜索方法

### 1. 重排序（Reranking）

在初步搜索结果上，使用更精确的模型重新排序。

```python
# 第一阶段：快速召回（Top-100）
candidates = client.search(
    collection_name="documents",
    data=[query_vector],
    anns_field="vector",
    limit=100
)

# 第二阶段：精确排序（Top-10）
from sentence_transformers import CrossEncoder
reranker = CrossEncoder('cross-encoder/ms-marco-MiniLM-L-6-v2')

scores = reranker.predict([
    (query_text, hit['entity']['text']) 
    for hit in candidates[0]
])

# 根据新分数排序
reranked = sorted(
    zip(candidates[0], scores), 
    key=lambda x: x[1], 
    reverse=True
)[:10]
```

### 2. 多阶段检索

结合多种搜索方式，逐步精炼结果。

```python
# 阶段 1：粗召回（向量搜索）
stage1 = client.search(..., limit=1000)

# 阶段 2：过滤（元数据）
stage2 = [hit for hit in stage1 if hit['entity']['quality'] > 0.8]

# 阶段 3：重排序（精确模型）
stage3 = rerank(stage2, query)

# 阶段 4：多样性（去重、分组）
final = diversify(stage3)
```

### 3. 负样本搜索

找到与查询向量不相似的结果。

```python
# 使用 IP 或 COSINE，取分数最低的
results = client.search(
    collection_name="documents",
    data=[query_vector],
    anns_field="vector",
    limit=10,
    search_params={"metric_type": "IP"}
)

# 反转结果顺序
negative_results = results[0][::-1]
```

### 4. 批量相似度计算

计算多个向量之间的两两相似度。

```python
# 获取所有向量
all_vectors = client.query(
    collection_name="documents",
    filter="",
    output_fields=["id", "vector"],
    limit=1000
)

# 计算相似度矩阵
import numpy as np
vectors = np.array([v['vector'] for v in all_vectors])
similarity_matrix = np.dot(vectors, vectors.T)
```

## 搜索性能优化

# 调优索引参数

)

## 如何选择搜索方法？

### 决策树

```
开始
 ↓
需要精确关键词匹配？
 ├─ 是 → 使用 BM25 全文搜索
 └─ 否 ↓
    需要大量结果（>1000）？
     ├─ 是 → 使用迭代搜索
     └─ 否 ↓
        需要多样性（避免单一来源）？
         ├─ 是 → 使用分组搜索
         └─ 否 ↓
            需要元数据过滤？
             ├─ 是 → 使用过滤搜索
             └─ 否 ↓
                需要距离阈值判断？
                 ├─ 是 → 使用范围搜索
                 └─ 否 → 使用基本 ANN 搜索
```

### 场景推荐表

| 场景 | 推荐方法 | 度量方式 | 补充策略 |
|-----|---------|---------|---------|
| 语义搜索 | 基本 ANN | COSINE | + 重排序 |
| 电商搜索 | 过滤搜索 | L2 | + 价格/评分过滤 |
| RAG 系统 | 分组搜索 | COSINE | + 每文档多片段 |
| 文档检索 | BM25 + 向量 | COSINE | 混合搜索 |
| 人脸识别 | 范围搜索 | L2 | 距离阈值 |
| 推荐系统 | 基本 ANN | IP | + 协同过滤 |
| 图像检索 | 基本 ANN | L2 | + 特征提取 |
| 数据导出 | 迭代搜索 | - | 分批处理 |
| 精确查找 | 元数据查询 | - | Get/Query |

## 总结

### 核心要点

1. **度量方式选择**
   - 文本/语义 → COSINE
   - 图像/数值 → L2
   - 推荐系统 → IP

2. **搜索方法选择**
   - 通用场景 → 基本 ANN
   - 需要过滤 → 过滤搜索
   - 需要多样性 → 分组搜索
   - 关键词匹配 → BM25
   - 大量结果 → 迭代搜索

3. **性能优化**
   - 使用批量搜索
   - 只返回需要的字段
   - 在数据库层过滤
   - 选择合适的索引

4. **混合策略**
   - 向量 + 元数据过滤（最常用）
   - 向量 + BM25（效果最好）
   - 粗召回 + 精排序（性能最优）

记住：没有完美的搜索方法，根据具体场景选择最合适的组合！
