# Milvus 动态字段详解

## 问题：id 和 vector 也没定义，为什么不是动态字段？

### 答案：快速模式自动定义了 id 和 vector

```python
# 你写的代码
client.create_collection(
    collection_name="quick_setup",
    dimension=5  # 只指定了向量维度
)

# Milvus 实际做的事（自动生成 Schema）
from pymilvus import CollectionSchema, FieldSchema, DataType

fields = [
    # ⭐ 自动创建 id 字段（主键）
    FieldSchema(
        name="id",
        dtype=DataType.INT64,
        is_primary=True,
        auto_id=False  # 需要手动提供 id
    ),
    # ⭐ 自动创建 vector 字段
    FieldSchema(
        name="vector",
        dtype=DataType.FLOAT_VECTOR,
        dim=5  # 你指定的维度
    )
]

schema = CollectionSchema(
    fields=fields,
    enable_dynamic_field=True  # ⭐ 自动开启动态字段
)
```

## 完整流程演示

### 步骤1：创建集合（快速模式）

```python
client.create_collection(
    collection_name="quick_setup",
    dimension=5
)
```

**Milvus 内部操作**：
```python
# 1. 自动创建 Schema
fields = [
    FieldSchema(name="id", dtype=DataType.INT64, is_primary=True),
    FieldSchema(name="vector", dtype=DataType.FLOAT_VECTOR, dim=5)
]

# 2. 开启动态字段
schema = CollectionSchema(
    fields=fields,
    enable_dynamic_field=True  # ← 关键！
)

# 3. 创建集合
create_collection_with_schema(schema)
```

### 步骤2：查看 Schema

```python
info = client.describe_collection(collection_name="quick_setup")

print("Schema 中定义的字段：")
for field in info['fields']:
    print(f"  - {field['name']}: {field['type']}")

print(f"\n是否支持动态字段: {info['enable_dynamic_field']}")
```

**输出**：
```
Schema 中定义的字段：
  - id: DataType.INT64        ← 固定字段（自动创建）
  - vector: DataType.FLOAT_VECTOR  ← 固定字段（自动创建）

是否支持动态字段: True  ← 可以添加额外字段
```

### 步骤3：插入数据

```python
data = [
    {
        "id": 0,           # ← 固定字段（必须提供）
        "vector": [...],   # ← 固定字段（必须提供）
        "color": "red",    # ← 动态字段（可选）
        "price": 99.9      # ← 动态字段（可选）
    }
]
```

## 对比三种模式

### 模式1：快速模式（自动 Schema + 动态字段）

```python
# 代码
client.create_collection(
    collection_name="quick_setup",
    dimension=5
)

# 实际 Schema
固定字段：id, vector
动态字段：✅ 支持（任何字段都可以）
```

**插入数据**：
```python
data = [
    {
        "id": 0,           # 固定字段
        "vector": [...],   # 固定字段
        "color": "red",    # ✅ 动态字段
        "price": 99.9,     # ✅ 动态字段
        "anything": "ok"   # ✅ 任何字段都可以
    }
]
```

---

### 模式2：自定义模式（手动 Schema + 关闭动态字段）

```python
# 代码
fields = [
    FieldSchema(name="id", dtype=DataType.INT64, is_primary=True),
    FieldSchema(name="vector", dtype=DataType.FLOAT_VECTOR, dim=5),
    FieldSchema(name="color", dtype=DataType.VARCHAR, max_length=50),
    FieldSchema(name="price", dtype=DataType.FLOAT)
]

schema = CollectionSchema(
    fields=fields,
    enable_dynamic_field=False  # ← 关闭动态字段
)

client.create_collection(
    collection_name="custom_setup",
    schema=schema
)

# 实际 Schema
固定字段：id, vector, color, price
动态字段：❌ 不支持
```

**插入数据**：
```python
data = [
    {
        "id": 0,           # ✅ 固定字段
        "vector": [...],   # ✅ 固定字段
        "color": "red",    # ✅ 固定字段
        "price": 99.9,     # ✅ 固定字段
        "tag": "hot"       # ❌ 错误！不允许额外字段
    }
]
# 报错：Attempt to insert an unexpected field `tag`
```

---

### 模式3：混合模式（手动 Schema + 开启动态字段）⭐ 推荐

```python
# 代码
fields = [
    FieldSchema(name="id", dtype=DataType.INT64, is_primary=True),
    FieldSchema(name="vector", dtype=DataType.FLOAT_VECTOR, dim=5),
    FieldSchema(name="color", dtype=DataType.VARCHAR, max_length=50),
    FieldSchema(name="price", dtype=DataType.FLOAT)
]

schema = CollectionSchema(
    fields=fields,
    enable_dynamic_field=True  # ← 开启动态字段
)

client.create_collection(
    collection_name="hybrid_setup",
    schema=schema
)

# 实际 Schema
固定字段：id, vector, color, price（可以创建索引）
动态字段：✅ 支持（额外的灵活性）
```

**插入数据**：
```python
data = [
    {
        "id": 0,           # ✅ 固定字段
        "vector": [...],   # ✅ 固定字段
        "color": "red",    # ✅ 固定字段
        "price": 99.9,     # ✅ 固定字段
        "tag": "hot",      # ✅ 动态字段
        "stock": 100       # ✅ 动态字段
    }
]
```

## 固定字段 vs 动态字段的区别

| 特性 | 固定字段 | 动态字段 |
|------|---------|---------|
| 定义位置 | Schema 中明确定义 | 插入时随意添加 |
| 类型检查 | 严格 | 宽松 |
| 创建索引 | ✅ 可以 | ❌ 不可以 |
| 过滤搜索 | ✅ 可以 | ✅ 可以 |
| 性能 | 更快 | 稍慢 |

## 实际例子：电商商品搜索

### 场景：商品数据结构不统一

```python
# 商品1：手机
{
    "id": 1,
    "vector": [...],
    "name": "iPhone 15",
    "price": 5999,
    "color": "黑色",      # 手机特有
    "storage": "256GB"    # 手机特有
}

# 商品2：衣服
{
    "id": 2,
    "vector": [...],
    "name": "T恤",
    "price": 99,
    "color": "白色",      # 衣服特有
    "size": "L"           # 衣服特有
}

# 商品3：书籍
{
    "id": 3,
    "vector": [...],
    "name": "三体",
    "price": 45,
    "author": "刘慈欣",   # 书籍特有
    "pages": 500          # 书籍特有
}
```

### 解决方案：混合模式

```python
# 定义所有商品的共同字段
fields = [
    FieldSchema(name="id", dtype=DataType.INT64, is_primary=True),
    FieldSchema(name="vector", dtype=DataType.FLOAT_VECTOR, dim=128),
    FieldSchema(name="name", dtype=DataType.VARCHAR, max_length=200),
    FieldSchema(name="price", dtype=DataType.FLOAT)
]

schema = CollectionSchema(
    fields=fields,
    enable_dynamic_field=True  # ← 允许每个商品有自己的特殊字段
)

# 为共同字段创建索引（提高搜索速度）
client.create_index(
    collection_name="products",
    field_name="price",
    index_type="INVERTED"
)

# 搜索：价格在 50-100 之间的商品
results = client.search(
    collection_name="products",
    data=[query_vector],
    filter="price >= 50 and price <= 100",  # 使用固定字段过滤
    output_fields=["*"]  # 返回所有字段（包括动态字段）
)
```

## 总结

### 快速模式的完整流程

```python
# 1. 创建集合
client.create_collection(
    collection_name="quick_setup",
    dimension=5
)

# 2. Milvus 自动做的事
"""
自动创建 Schema：
  - id (INT64, 主键)        ← 固定字段
  - vector (FLOAT_VECTOR)   ← 固定字段
  
自动开启动态字段：
  - enable_dynamic_field = True
"""

# 3. 插入数据
data = [
    {
        "id": 0,           # 固定字段（必须）
        "vector": [...],   # 固定字段（必须）
        "color": "red",    # 动态字段（可选）
        "price": 99.9      # 动态字段（可选）
    }
]
```

### 记忆口诀

```
快速模式 = 自动创建 id + vector + 开启动态字段

固定字段（Schema 中定义）：
  - 必须提供
  - 可以创建索引
  - 性能更好

动态字段（enable_dynamic_field=True）：
  - 可选提供
  - 不能创建索引
  - 灵活性高
```

### 推荐使用场景

| 场景 | 推荐模式 | 原因 |
|------|---------|------|
| 快速原型 | 快速模式 | 简单快速 |
| 数据结构统一 | 自定义模式（关闭动态字段） | 性能最好 |
| 数据结构不统一 | 混合模式（开启动态字段） | 灵活 + 性能 |
| 生产环境 | 混合模式 | 两全其美 ⭐ |
