## 💡 使用建议

### 文档阅读顺序

**推荐：** 先看子模块文档，再看综合指南

```
快速开始.md (安装环境)
    ↓
查询翻译技术详解.md (学习具体技术)
查询路由技术详解.md
查询构建技术详解.md
    ↓
检索前处理技术综合指南.md (全面了解)
```

**为什么这样安排？**
- 子模块文档更短（每个约 3000-5000 字）
- 聚焦单一主题，易于理解
- 综合指南很长（约 15000 字），适合最后系统学习

### 代码运行顺序

1. **查询重写**（最简单）
   ```bash
   python 02-查询翻译/01-查询重写-1-通过提示词重写.py
   ```

2. **查询路由**
   ```bash
   python 03-查询路由/01-逻辑路由.py
   ```

3. **查询分解**（需要向量数据库）
   ```bash
   python 02-查询翻译/02-查询分解-1-MultiQueryRetriever.py
   ```

4. **HyDE**
   ```bash
   python 02-查询翻译/04-查询扩展-HyDE假设文档生成.py
   ```

5. **Text-to-SQL**
   ```bash
   python 01-查询构建/Text2SQL/02-Text2SQL-LLM-DeepSeek.py
   ```

## ❓ 常见问题

### Q: 文档太长，从哪里开始？

**A:** 不需要。根据你想使用的功能选择性安装。最小化安装只需要：

### Q: 代码运行报错怎么办？
