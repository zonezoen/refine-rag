## 前言

前面我们学习了重排和压缩技术，知道如何提升检索精度和降低成本。但还有一个关键问题：如何保证检索结果的质量？传统RAG系统往往盲目信任检索结果，当知识库信息不完整或过时时，就会产生错误答案。

校正技术，特别是CRAG（Corrective Retrieval Augmented Generation），为RAG系统增加了"自我反思"能力，能够主动检测和修正检索错误，确保答案的准确性和可靠性。

## 什么是校正（Correction）？

### 用生活例子理解校正

想象你在图书馆找资料：

传统方法（盲目信任）：
```
1. 你问管理员："五台山的开放时间？"
2. 管理员给你一本书："五台山旅游指南"
3. 你直接用这本书写答案
4. 结果：书里根本没有开放时间信息！
```

CRAG方法（自我校正）：
```
1. 你问管理员："五台山的开放时间？"
2. 管理员给你一本书："五台山旅游指南"
3. 你先检查：这本书有开放时间信息吗？
4. 发现：没有！这本书主要讲历史
5. 你重新问："五台山景区营业时间"
6. 管理员给你另一本书，这次有了！
```

校正就是这个"检查-重试"的过程！

## 为什么需要校正？

### 传统RAG的三大问题

#### 问题1：检索不准确

```
用户问："iPhone 15的价格是多少？"

检索结果：
- 文档1："iPhone 14的价格是5999元"
- 文档2："iPhone的发展历史..."
- 文档3："苹果公司简介..."

传统RAG：
→ 直接用这些文档生成答案
→ 答案：根据检索结果，iPhone价格是5999元（错误！）

CRAG：
→ 检测到文档不相关
→ 重写查询："iPhone 15 价格 2024"
→ 网络搜索获取最新信息
→ 答案：iPhone 15价格是5999元起（正确！）
```

#### 问题2：信息过时

```
用户问："今天北京的天气怎么样？"

知识库文档：
- "2023年12月1日，北京晴天，温度5°C"（过时）

传统RAG：
→ 返回过时信息

CRAG：
→ 检测到信息可能过时
→ 触发实时搜索
→ 返回今天的天气
```

#### 问题3：知识库不完整

```
用户问："最新的GPT-5有哪些功能？"

知识库：
- 只有GPT-4的信息

传统RAG：
→ 用GPT-4的信息回答GPT-5的问题（错误）

CRAG：
→ 检测到知识库没有相关信息
→ 网络搜索补充
→ 返回准确答案或说明"暂无信息"
```

## CRAG（反思式检索）详解

**文件：** `01-CRAG-反思式检索.py`

### 核心思想

让RAG系统具备"自我反思"能力：
1. 检索后先评估质量
2. 如果质量不好，主动修正
3. 必要时寻求外部帮助（网络搜索）

### CRAG完整工作流程

```
用户提问
    ↓
【步骤1：初步检索】
向量数据库检索
    ↓
【步骤2：质量评估】核心创新
LLM评估每个文档的相关性
    ↓
【步骤3：决策分支】
    ├─ 所有文档相关 → 直接生成答案
    ├─ 部分文档相关 → 过滤后生成答案
    └─ 没有相关文档 → 查询重写 + 网络搜索
         ↓
    【步骤4：校正】
    重写查询 + 网络搜索
         ↓
    【步骤5：生成答案】
    基于新信息生成
```

#### 步骤2：质量评估（CRAG的核心）

```python
# 定义评分模型
class GradeDocuments(BaseModel):
    binary_score: str = Field(
        description="文档相关为'yes'，不相关为'no'"
    )

# 创建评分器
llm = ChatOpenAI(model="gpt-4o", temperature=0.5)
grader = llm.with_structured_output(GradeDocuments)

# 评分提示
system = """你是一个评估检索文档与用户问题相关性的评分员。
如果文档包含与问题相关的关键词或语义含义，则将其评为相关。
给出一个二元评分'yes'或'no'。"""

# 对每个文档评分
for doc in documents:
    score = grader.invoke({
        "question": query,
        "document": doc.page_content
    })
    print(f"文档相关性: {score.binary_score}")
```

**评分示例：**
```
查询："五台山的开放时间？"

文档1："五台山位于山西省..." 
→ 评分：no（只有位置信息，没有开放时间）

文档2："五台山全年开放，6:00-18:00"
→ 评分：yes（直接回答了问题）

文档3："五台山是佛教名山..."
→ 评分：no（讲历史，不相关）
```

#### 步骤3：决策逻辑

```python
def decide_to_generate(state):
    """
    决策函数：根据评分结果选择下一步
    """
    web_search = state["web_search"]
    
    if web_search == "Yes":
        # 没有相关文档 → 需要校正
        print("决策: 转换查询并搜索")
        return "transform_query"
    else:
        # 有相关文档 → 直接生成
        print("决策: 直接生成答案")
        return "generate"
```

**决策树：**
```
评估结果
    ├─ 所有文档都相关（100%）
    │   → 直接生成答案
    │
    ├─ 部分文档相关（50-99%）
    │   → 过滤掉不相关的，用相关的生成
    │
    └─ 没有相关文档（0%）
        → 查询重写 + 网络搜索
```

## CRAG的核心优势

### 1. 自动质量控制

```
传统RAG：
检索 → 生成（盲目信任检索结果）

CRAG：
检索 → 评估 → 决策 → 生成（有质量保证）
```

### 2. 主动纠错

```
场景：知识库信息不完整

传统RAG：
→ 用不完整的信息生成答案（可能错误）

CRAG：
→ 检测到信息不足
→ 主动搜索补充
→ 生成准确答案
```

### 3. 多信息源融合

```
CRAG可以结合：
- 本地知识库（快速、私密）
- 网络搜索（最新、全面）
- 数据库查询（结构化数据）
- API调用（实时数据）
```

## CRAG的实现细节

### 状态管理（GraphState）

```python
from typing import List
from typing_extensions import TypedDict

class GraphState(TypedDict):
    """
    CRAG工作流的状态
    """
    question: str        # 当前问题（可能被重写）
    generation: str      # 生成的答案
    web_search: str      # 是否需要网络搜索（"Yes"/"No"）
    documents: List[str] # 文档列表
```

### 节点函数

```python
# 节点1：检索
def retrieve(state):
    question = state["question"]
    documents = retriever.get_relevant_documents(question)
    return {"documents": documents, "question": question}

# 节点2：评分
def grade_documents(state):
    question = state["question"]
    documents = state["documents"]
    
    filtered_docs = []
    web_search = "No"
    
    for doc in documents:
        score = grader.invoke({
            "question": question,
            "document": doc.page_content
        })
        
        if score.binary_score == "yes":
            filtered_docs.append(doc)
        else:
            web_search = "Yes"  # 有不相关文档，标记需要搜索
    
    return {
        "documents": filtered_docs,
        "question": question,
        "web_search": web_search
    }

# 节点3：查询转换
def transform_query(state):
    question = state["question"]
    better_question = question_rewriter.invoke({"question": question})
    return {"question": better_question}

# 节点4：网络搜索
def web_search(state):
    question = state["question"]
    search_results = web_search_tool.invoke(question)
    web_doc = Document(page_content=str(search_results))
    state["documents"].append(web_doc)
    return state

# 节点5：生成
def generate(state):
    question = state["question"]
    documents = state["documents"]
    generation = rag_chain.invoke({
        "context": documents,
        "question": question
    })
    return {"generation": generation}
```

### 工作流图构建

```python
from langgraph.graph import StateGraph, START, END

# 创建工作流
workflow = StateGraph(GraphState)

# 添加节点
workflow.add_node("retrieve", retrieve)
workflow.add_node("grade_documents", grade_documents)
workflow.add_node("generate", generate)
workflow.add_node("transform_query", transform_query)
workflow.add_node("web_search_node", web_search)

# 添加边
workflow.add_edge(START, "retrieve")
workflow.add_edge("retrieve", "grade_documents")

# 条件边：根据评分结果选择路径
workflow.add_conditional_edges(
    "grade_documents",
    decide_to_generate,
    {
        "transform_query": "transform_query",
        "generate": "generate"
    }
)

workflow.add_edge("transform_query", "web_search_node")
workflow.add_edge("web_search_node", "generate")
workflow.add_edge("generate", END)

# 编译
app = workflow.compile()
```

## 重要知识点

### 1. Self-RAG（自我反思RAG）

#### 什么是Self-RAG？

Self-RAG是CRAG的增强版，不仅在检索后反思，还在生成过程中持续反思。

#### Self-RAG vs CRAG

```
CRAG：
检索 → 评估一次 → 生成

Self-RAG：
检索 → 评估 → 生成第一句 → 评估 → 生成第二句 → 评估 → ...
（每生成一句都评估质量）
```

#### Self-RAG的反思点

```python
# 反思点1：是否需要检索？
def should_retrieve(query):
    """
    判断这个问题是否需要检索外部知识
    """
    # 简单问题：不需要
    if is_simple_question(query):
        return False
    
    # 需要事实的问题：需要
    if requires_facts(query):
        return True
    
    return False

# 反思点2：检索结果是否相关？
def is_relevant(doc, query):
    """
    评估文档相关性
    """
    score = relevance_scorer(doc, query)
    return score > threshold

# 反思点3：生成的内容是否有支撑？
def is_supported(generation, documents):
    """
    检查生成的内容是否有文档支撑
    """
    for claim in extract_claims(generation):
        if not has_support(claim, documents):
            return False
    return True

# 反思点4：生成的答案是否有用？
def is_useful(generation, query):
    """
    评估答案质量
    """
    return usefulness_scorer(generation, query) > threshold
```

#### Self-RAG工作流程

```python
def self_rag(query):
    """
    Self-RAG完整流程
    """
    # 反思1：需要检索吗？
    if should_retrieve(query):
        docs = retrieve(query)
        
        # 反思2：文档相关吗？
        relevant_docs = [d for d in docs if is_relevant(d, query)]
    else:
        relevant_docs = []
    
    # 生成答案（逐句）
    answer = ""
    for _ in range(max_sentences):
        sentence = generate_next_sentence(query, relevant_docs, answer)
        
        # 反思3：这句话有支撑吗？
        if not is_supported(sentence, relevant_docs):
            # 重新检索或重新生成
            continue
        
        answer += sentence
        
        # 反思4：答案足够了吗？
        if is_complete(answer, query):
            break
    
    # 反思5：最终答案有用吗？
    if not is_useful(answer, query):
        # 重新生成
        answer = regenerate(query, relevant_docs)
    
    return answer
```

### 2. 迭代检索（Iterative Retrieval）

#### 什么是迭代检索？

不是一次性检索，而是根据生成的内容多次检索。

#### 工作流程

```python
def iterative_retrieval(query, max_iterations=3):
    """
    迭代检索：逐步完善答案
    """
    answer = ""
    context = []
    
    for i in range(max_iterations):
        # 第一次：用原始查询检索
        # 后续：用"查询+已生成内容"检索
        search_query = query if i == 0 else f"{query} {answer}"
        
        # 检索
        new_docs = retrieve(search_query)
        context.extend(new_docs)
        
        # 生成
        answer = generate(query, context)
        
        # 检查是否完整
        if is_complete(answer):
            break
    
    return answer
```

#### 示例

```
查询："介绍一下云冈石窟"

第1次迭代：
- 检索："云冈石窟"
- 获得：基本信息
- 生成："云冈石窟位于山西大同..."

第2次迭代：
- 检索："云冈石窟 云冈石窟位于山西大同"
- 获得：历史背景
- 生成："云冈石窟位于山西大同，建于北魏时期..."

第3次迭代：
- 检索："云冈石窟 云冈石窟位于山西大同，建于北魏时期"
- 获得：艺术特色
- 生成："云冈石窟位于山西大同，建于北魏时期，以精美的佛教雕塑著称..."
```

### 3. 主动学习（Active Learning）

#### 什么是主动学习？

系统主动识别不确定的情况，请求人工标注。

#### 在CRAG中的应用

```python
def crag_with_active_learning(query):
    """
    带主动学习的CRAG
    """
    # 检索和评分
    docs = retrieve(query)
    scores = [grade(doc, query) for doc in docs]
    
    # 计算置信度
    confidence = calculate_confidence(scores)
    
    if confidence < threshold:
        # 置信度低：请求人工确认
        human_feedback = request_human_feedback(query, docs)
        
        # 用反馈更新模型
        update_grader(query, docs, human_feedback)
        
        # 重新评分
        scores = [grade(doc, query) for doc in docs]
    
    # 继续正常流程
    return generate_answer(query, docs, scores)
```

#### 主动学习的触发条件

```python
def should_request_feedback(scores):
    """
    判断是否需要人工反馈
    """
    # 条件1：所有分数都接近阈值（不确定）
    if all(0.4 < s < 0.6 for s in scores):
        return True
    
    # 条件2：分数分歧很大
    if max(scores) - min(scores) > 0.5:
        return True
    
    # 条件3：历史上这类查询错误率高
    if get_error_rate(query_type) > 0.3:
        return True
    
    return False
```

### 4. 多跳推理（Multi-hop Reasoning）

#### 什么是多跳推理？

需要多次检索和推理才能回答的复杂问题。

#### 示例

```
问题："五台山和云冈石窟哪个更早被列入世界遗产？"

需要：
1. 检索五台山的世界遗产时间
2. 检索云冈石窟的世界遗产时间
3. 比较两个时间
4. 生成答案
```

#### 实现方法

```python
def multi_hop_crag(query):
    """
    多跳CRAG
    """
    # 分解问题
    sub_questions = decompose_question(query)
    # ["五台山何时被列入世界遗产？", "云冈石窟何时被列入世界遗产？"]
    
    # 对每个子问题执行CRAG
    sub_answers = []
    for sub_q in sub_questions:
        answer = crag(sub_q)
        sub_answers.append(answer)
    
    # 综合答案
    final_answer = synthesize(query, sub_answers)
    return final_answer
```

### 5. 混合检索策略

#### 什么是混合检索？

结合多种检索方法，提高召回率。

#### 常见组合

```python
def hybrid_retrieval(query):
    """
    混合检索：向量检索 + BM25 + 知识图谱
    """
    # 方法1：向量检索（语义相似）
    vector_docs = vector_retriever.get_relevant_documents(query)
    
    # 方法2：BM25（关键词匹配）
    bm25_docs = bm25_retriever.get_relevant_documents(query)
    
    # 方法3：知识图谱（结构化知识）
    kg_docs = kg_retriever.get_relevant_documents(query)
    
    # 融合结果（使用RRF）
    all_docs = [vector_docs, bm25_docs, kg_docs]
    fused_docs = rrf_fusion(all_docs)
    
    return fused_docs
```

def crag_with_hybrid_retrieval(query):

    使用混合检索的CRAG
    """

    if len(relevant_docs) == 0:

        relevant_docs = grade_and_filter(docs, expanded_query)
        
        # 还是不行，网络搜索
        if len(relevant_docs) == 0:
            web_docs = web_search(query)
            relevant_docs = web_docs
    
    # 生成答案

#### 基于置信度的决策

```python
def confidence_based_decision(scores):
    """
    基于置信度的决策
    """
    high_confidence_relevant = [
        s for s in scores 
        if s.binary_score == "yes" and s.confidence > 0.8
    ]
    
    if len(high_confidence_relevant) > 0:
        # 有高置信度的相关文档：直接生成
        return "generate"
    
    low_confidence = [s for s in scores if s.confidence < 0.5]
    
    if len(low_confidence) > len(scores) * 0.5:
        # 超过一半的文档置信度低：重新检索
        return "re_retrieve"
    
    # 默认：查询重写
    return "transform_query"
```

### 7. 错误恢复机制

#### 什么是错误恢复？

当CRAG流程出错时，有备用方案。

#### 常见错误和恢复策略

```python
def crag_with_error_recovery(query):
    """
    带错误恢复的CRAG
    """
    try:
        # 尝试正常CRAG流程
        return crag(query)
    
    except RetrievalError:
        # 检索失败：使用缓存或默认答案
        print("检索失败，使用缓存")
        return get_cached_answer(query)
    
    except GradingError:
        # 评分失败：跳过评分，直接生成
        print("评分失败，跳过评分")
        docs = retrieve(query)
        return generate(query, docs)
    
    except WebSearchError:
        # 网络搜索失败：只用本地知识
        print("网络搜索失败，只用本地知识")
        docs = retrieve(query)
        relevant_docs = simple_filter(docs, query)
        return generate(query, relevant_docs)
    
    except GenerationError:
        # 生成失败：返回检索结果
        print("生成失败，返回检索结果")
        docs = retrieve(query)
        return format_documents(docs)
```

def crag_with_fallback(query):

#### 1. 准确率（Accuracy）

```python
def evaluate_accuracy(test_cases):
    """
    评估CRAG的准确率
    """
    correct = 0
    total = len(test_cases)
    
    for query, ground_truth in test_cases:
        answer = crag(query)
        if is_correct(answer, ground_truth):
            correct += 1
    
    return correct / total
```

#### 2. 召回率（Recall）

```python
def evaluate_recall(test_cases):
    """
    评估是否找到了所有相关信息
    """
    total_relevant = 0
    found_relevant = 0
    
    for query, relevant_docs in test_cases:
        retrieved_docs = crag_retrieve(query)
        
        total_relevant += len(relevant_docs)
        found_relevant += len(set(retrieved_docs) & set(relevant_docs))
    
    return found_relevant / total_relevant
```

#### 3. 校正率（Correction Rate）

```python
def evaluate_correction_rate(test_cases):
    """
    评估CRAG的校正能力
    """
    corrections = 0
    total = len(test_cases)
    
    for query, _ in test_cases:
        # 记录是否触发了校正
        if triggered_correction(query):
            corrections += 1
    
    return corrections / total
```

    if relevance_ratio == 0:

```python
def cost_aware_crag(query, budget):
    """
    成本感知的CRAG
    """
    # 第一步：免费的本地检索
    docs = retrieve(query)
    
    if budget > 0.1:
        # 有预算：使用LLM评分（准确但贵）
        scores = llm_grade(docs, query)
    else:
        # 无预算：使用简单评分（快速但不够准）
        scores = simple_grade(docs, query)
    
    relevant_docs = filter_by_scores(docs, scores)
    
    if len(relevant_docs) == 0 and budget > 0.05:
        # 需要搜索且有预算
        web_docs = web_search(query)
        relevant_docs = web_docs
    
    return generate(query, relevant_docs)
```
